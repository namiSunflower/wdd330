Responsive retrofitting- Taking what you already have and making it work on mobile.

Pros:

  1. Cheaper and faster than a mobile-first rewrite
  2. Better mobile experience than a desktop only site

Cons:

  1. Doesn't address screen real estate issues
  2. Doesn't address touch-interacation issues

Mobile First Approach

Pros:

  1. Provides best experience across many devices
  2. Address constraints of monile size/bandwidth
  3. Future friendly and ready for growth

Cons:

  1. Time consuming and costly for a full rebuild

Making Forms More User-Friendly

  1. Use float labels
  2. Reduce fields
  3. Paginate the form and display a progress bar if many fields are required
  4. Enable autocomplete
  5. Don't hide passwords by default but allows users to enable it easily
  6. Place visual indicators
  7. Show number keypad instead of default keyboard when taking for number input

Loading a Website Faster

  1. Reduce image usage throughout site/apps
  2. Use SVG/Webfonts since they are lower bandwidth
  3. Optimize and minify CSS and JS files
  4. GZIP files to eliminate unnecessary data

Effective Placing of Ads on Mobile

  1. Ads shouldn't take up 40% or more of the screen
  2. Stop using pop-ups and pop-overs
  3. Keep it simple and sleek
  4. Make sure it's unobtrusive

Additional notes:

  1. Start small and scale up.
  2. Mobile is also not a fad.
  3. Most people don't know what they want until you show it to them.
  4. Reduce user's cognitive load by making interactions simple, smooth, and seamless.
  5. Put navbar bottom on mobile to address reachability issues.
