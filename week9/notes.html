Browser Object Model

The Browser Object Model (or BOM for short) is a collection of properties and methods that contain information about the browser and computer screen.

Window Dialog Method/s

  1. window.confirm(): will stop the execution of the program and display a confirmation dialog that shows the message provided as an argument, and giving the options of OK or Cancel.
  2. window.prompt(): displays a dialog that shows a message provided as an argument, as well as an input field that allows the user to enter text.

Example Syntax (window.confirm):

window.confirm('Do you wish to continue?');

Example Syntax (window.prompt):

window.prompt('Please enter your dog's name:');

Window Browser & Location Info Method/s

  1. window.navigator.userAgent: the userAgent property will return information about the browser and operating system being used.
  2. window.location.href: contains a number of properties that provide information about different fragments of the URL of the current page.
  3. window.location.protocol: returns a string describing the protocol used (such as http, https, pop2, ftp etc.).
  4. window.location.host: returns a string describing the domain of the current URL and the port number (often omitted if the default port 80 is used).
  5. window.location.hostname: returns a string describing the domain of the current URL.
  6. window.location.port: returns a string describing the port number, although it will return an empty string if the port is not explicitly stated in the URL.
  7. window.location.search: returns a string that starts with a “?” followed by the query string parameters.
  8. window.navigator.pathname: returns a string of the path that follows the domain.
  9. window.location.hash: returns a string that starts with a “#” followed by the fragment identifier. It returns an empty string if there is no fragment identifier.
  10. window.location.origin: returns a string that shows the protocol and domain where the current page originated from.
  11. window.location.toString(): returns a string containing the whole URL

Window Loading/Reloading Method/s

  1. window.location.reload: can be used to force a reload of the current page. If it’s given a parameter of true, it will force the browser to reload the page from the server, instead of using a cached page.
  2. window.location.assign(): can be used to load another resource from a URL provided as a parameter
  3. window.location.replace(): almost the same as the assign() method, except the current page will not be stored in the session history, so the user will be unable to navigate back to it using the back button.

Window Browser History Method/s

This property can be used to access information about any previously visited pages in the current browser session

  1. window.history.length: shows how many pages have been visited before arriving at the current page.
  2. window.history.go: can be used to go to a specific page, where 0 is the current page.

Example Syntax (history.go):

window.history.go(1); // goes forward 1 page
window.history.go(0); // reloads the current page
window.history.go(-1); // goes back 1 page

Controlling the Window Method/s

  1. window.open(URL, name, specs, replace): shows how many pages have been visited before arriving at the current page.
  2. window.close(): can be used to close a window, assuming you have a reference to it.
  3. window.resizeTo(width, height): takes two parameters that specify the width and height of the resized window’s dimensions.

Window Screen Info Method/s

  1. window.screen.height: return height of screen in pixels.
  2. window.screen.height: return width of screen in pixels
  3. window.screen.availWidth: return width of screen in pixels, excluding any operating system menus.
  4. window.screen.availHeight: return height of screen in pixels, excluding any operating system menus.
  5. window.screen.colorDepth: can be used to find the color bit depth of the user’s monitor, although there are few use cases for doing this other than collecting user statistics.

Window Timing Method/s

  1. window.setTimeout(function, milliseconds): accepts a callback to a function as its first parameter and a number of milliseconds as its second parameter.;
  2. window.clearTimeout(function): cancels timeout.
  3. window.setInterval(function, milliseconds): works in a similar way to window.setTimeout(), except that it will repeatedly invoke the callback function after every given number of milliseconds..
  4. window.clearInterval(function): cancels interval.

Window Animation Method/s

  1. window.requestAnimationFrame(callback): works in much the same way as the window.setInterval() method, although it has a number of improvements to optimize its performance. These include making the most of the browser’s built-in graphics-handling capabilities, and not running the animation when the tab is inactive, resulting in a much smoother performance.

Example Syntax (requestAnimationFrame):

const squareElement = document.getElementById('square');
let angle = 0;

function rotate() {
angle = (angle + 2)%360;
squareElement.style.transform = `rotate(${angle}deg)`
window.requestAnimationFrame(rotate);
}

const id = requestAnimationFrame(rotate);
