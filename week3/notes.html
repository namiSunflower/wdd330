Two Ways to Create an Array in JS:

  1. const myArray =[]
  2. const myArray = new Array();

It's safer to use array literals or "[]" than new Array() because we can override the value of Array in JavaScript:

Example:

Array = function() {};
var x = new Array();
//x is now an Object instead of an Array.

An array can also contain different types of data.

Example:

const mixedArray = [null, 1, [], 'two', true];

Destructuring Arrays-it's the concept of taking values out of an array and presenting them as individual values. Destructuring also allows us to assign multiple values at the same time using arrays:

const [x, y] = [1,2];

Even if the assignment was made using array, each individual variable still exists on its own outside the array. And we can see the results by checking the value of each variable:

  1. x=1
  2. y=2

Destructuring also allows us to swap the value of two variables over:

[x, y] = [y, x]

Results:

  1. x=2
  2. y=1

Viewing the value of the last element in an array:

const avengers = ['Captain America', 'Iron Man', 'Thor', 'Hulk', 'Hawkeye', 'Black Widow']
console.log(avengers[avengers.length -1];
Result: Black Widow

To include a new value or element at the beginning of an array, we can use the .unshift() method.

Syntax:

arrayName.unshift(newElement);

Finding if an array contains a specific element:

  1. arrayName.indexOf(element);
  2. arrayName.includes(element):

If we run the first code above when finding out if an array contains a specific element, it will return the index of that element if the element exists. If the item is not in the array, it will return -1 instead.

The .includes() method will return either true or false to show if the element does/does not exist within the array.

We can also add an extra parameter to indicate which index to start the search from:

arrayname.includes(element, index) //will start the search from the index value specified within the parentheses.

Join method:

It can he used to an array into a string that comprises all the items in the array separated by commas.

Example:

letters.join()
Result = 'a, b, c, d, e, f'

We can also choose another separator other than a comma by placing it inside the parentheses:

numbers.join('&')
Result = '1 & 2 & 3 & 4'

Slicing & Splicing:

Slice method- creates a subarray and chops out a slice of the original array.

Example:

avengers.slice(2,4) = ['Thor', 'Hulk']

Note that this method is non-destructive and no items are actually removed from the array.

Splice method- removes items from an array and inserts new items in their place.

Example:

avengers.splice(3, 1, 'Scarlet Witch')

The code above will remove whatever element is in the third index value and replace it with Scarlet Witch. The number next to 3 from the example determines how many items will be removed starting from the third index value.

Looping Over a Set:

const letters = new Set('hello');
for(const letter of letters) {
console.log(letter);
}
Result: 1, 2, 3, 4, 5

Looping Over a Map’s Values:

for(const value of RomanNumerals.values()) {
console.log(value);
}
Result: I, II, III, IV, V

Accessing Both Keys & Maps Using entries() method:

for(const [key,value] of RomanNumerals.entries()) {
console.log(`${key} in Roman numerals is ${value}`);
}
Result: 1 in Roman numerals is I, 2 in Roman numerals is II, 3 in Roman numerals is III, 4 in Roman numerals is IV, 5 in Roman numerals is V

Accessing Properties & Methods of Objects:

  1. Dot-notation: superman.name
  2. Bracket notation: superman[“name”]
  3. Dot-notation: superman.fly()
  4. Bracket notation: superman[“fly”]()

Although dot-notation is more commonly used, bracket notation comes in handy when trying to access non-standard property and method names.

Example:

superman["real" + " " + "name"] // the property is built using string concatenation
Result: "Clark Kent"

The parse() method takes a string of data in JSON format and returns a JavaScript object:

JSON.parse(batman);
Result:
{ name: 'Batman',
'real name': 'Bruce Wayne',

weight: 210,

villain: false,

The stringify() method does the opposite, taking a JavaScript object and returning a string of JSON data.
