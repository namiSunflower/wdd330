Terminology:

Automatic Semicolon Insertion- a process in JS that automatically places semicolons at the end of lines. But, it can be error-prone, so it's best practice to write JS statements line-by-line and add semicolons at the end of each line.

Block- series of statements collected together inside of curly brackets.

typeof- special JS operator to find a type of the value.

Type coercion- This happens when JavaScript attempts to convert one operand to an equivalent value of the other operand's type.

Data Types & Variables:

Non-primitive data types are arrays, functions, or objects.

Variables that start with an underscore generally refer to private properties and methods.

When assigning a primitive value to a variable, any changes made will only directly affect that value.

Example:

const a= 1

let b = a; // a =1, b = 1

b = 2: // a =1, b = 2

But when assigning non-primitive values to a variable, then it is done by reference. And any changes made will affect all references to that object.

Example:

const c = { value: 1};

let d = c; //c.value = 1, d.value = 1

d.value = 2; // c.value = 2, d.value = 2

Valid ways to put an apostrophe in a string:

  1. "It's me"
  2. 'It\'s me'
  3. It's me

Template literals allow interpolation of JS code. It means that a JS expression can be inserted inside a string and the result will be displayed, as can be seen in the example below:

const name = Siri;

Hello ${ name }!;

=='Helo Siri!'

Type Coercion Example:

'2' * 1 = 3

But if we try and add a string and a number together, it will just concatenate them as a single string.

Example:

'2' + 1 = 21

Modulo Operator:

When using the modulo operator, the result will keep the sign of the first number.

Example:

-4%3 = -1

Arrow Function Syntax:

const add = (x,y) => x + y;
